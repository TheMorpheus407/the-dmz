## Summary

In `apps/web/src/lib/game/components/InboxList.svelte` at line 266, the `<ul>` element incorrectly has `role="option"` applied to it:

```svelte
<ul class="email-list" role="option">
```

This violates the ARIA listbox pattern. The `role="option"` should only be on the `<li>` child elements (line 277), not on the `<ul>` container.

## Impact

- **Screen reader confusion**: Screen readers will announce the entire `<ul>` as a single option, making the listbox inaccessible to users of assistive technology
- **Broken navigation**: Users relying on listbox navigation patterns (arrow keys, etc.) will not be able to properly navigate individual email items
- **Incorrect semantics**: The ARIA listbox pattern expects options to be direct children with `role="option"`, not the list container

## Evidence

File: `apps/web/src/lib/game/components/InboxList.svelte`
Line: 266

```svelte
<ul class="email-list" role="option">   <!-- WRONG: role="option" on ul -->
  {#each items as item (item.email.emailId)}
    <li
      class="email-item"
      ...
      role="option"                         <!-- CORRECT: role="option" on li -->
      aria-selected={isSelected(email.emailId)}
      ...
    >
```

The correct ARIA listbox structure should be:

- Container: `role="listbox"` (line 243: `<div class="inbox-list" role="listbox" ...>`)
- Options: Each `<li>` with `role="option"` (already correctly implemented on line 277)
- The `<ul>` should NOT have any role, or could use `role="presentation"` to hide its semantics

## Recommended Fix

Remove `role="option"` from the `<ul>` element at line 266. The correct markup should be:

```svelte
<ul class="email-list">
```

The `<li>` elements already correctly have `role="option"` and `aria-selected`, which is all that is needed for the listbox pattern.
